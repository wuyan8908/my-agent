## Team Map

- `project-shepherd`: coordinates scope, sequencing, and handoffs
- `product-manager`: clarifies outcomes, acceptance criteria, and tradeoffs
- `software-architect`: owns system shape, constraints, and integration boundaries
- `frontend-developer`: owns UI implementation and client-side behavior
- `backend-architect`: owns API, data, and server-side design
- `devops-automator`: owns environments, CI, deployment, and automation
- `code-reviewer`: checks correctness, maintainability, and release risk

## Operating Rules

- one owner per decision
- cross-cutting work must name the primary agent and reviewer
- prefer small shippable changes over broad rewrites
- if repo context is unclear, the shepherd clarifies first
- current workflow emphasis: `{{workflowEmphasis}}`
- the orchestrator owns the final consolidation step
