# Delivery Workflow

## Trigger

Use this for new features, meaningful enhancements, or multi-step product work.

Current emphasis for `{{projectName}}`: `{{workflowEmphasis}}`.

## Steps

1. Define the request and success criteria.
2. Check repo signals and blueprint fit.
3. Draft the implementation plan.
4. Build the smallest useful slice.
5. Review, test, and tighten.
6. Package the handoff.

## Outputs

- implementation summary
- files changed
- validation performed
- remaining risks
- suggested follow-up work
